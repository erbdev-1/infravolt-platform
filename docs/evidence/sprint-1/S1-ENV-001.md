## Implementation

- Added exact runtime dependency `zod@4.4.3`.
- Added native `URL` parsing for HTTP(S) origins, lowercase/trailing-dot host normalization, default-port normalization and rejection of credentials, paths, queries and fragments.
- Added separate public and server parsers with explicit input selection and mandatory environment mode for client fixture/parser calls.
- Added `EnvironmentValidationError`, which stores and prints variable names only. It does not retain Zod issues or input values.
- Added production build/server validation through Next.js production configuration phases using the actual client and server readers.
- Added omission of optional keys after empty strings normalize to `undefined`.
- Added server-only semantic validation for plain email addresses and `Display Name <address>` mailboxes, including control-character rejection.
- Added a safe `.env.example` with the complete approved inventory, required/optional comments and only the two approved local site origins as non-empty examples.
- Added `verify:env`, a narrow TypeScript fixture runner compiled to ignored `.next` output. It exists because FND-023/WP-15 has not yet introduced Vitest and does not establish a competing test framework.
